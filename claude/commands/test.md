Genera tests para el archivo o función especificada.

## Casos a cubrir
- Happy path (funcionamiento normal)
- Edge cases (valores límite, vacíos, null)
- Error cases (inputs inválidos)

## Estructura
- Usa el framework de testing del proyecto
- Agrupa tests por función/método
- Sigue patrón Arrange → Act → Assert

Detecta automáticamente el framework de testing del proyecto antes de generar.
